01

背景引入

数据处理的发展历程

  • 手工时代 — Excel、脚本处理，效率低、易出错
  • 数据仓库时代 — ETL工具、Hadoop生态，批处理为主
  • 机器学习时代 — 特征工程、标注工具，数据质量开始重要
  • 大模型时代 — 高质量数据、RAG检索，数据成为核心资产

从"数量"转向"质量"
从"结构化"转向"多模态"

数据处理重心随AI演进不断变化
大模型时代，数据质量直接决定模型效果

大模型时代的数据挑战

数据质量决定模型效果

大模型训练遵循"垃圾进，垃圾出"原则，低质量数据导致模型输出质量差、幻觉问题严重

数据准备周期漫长

从数据采集到可用于训练，往往需要数周甚至数月时间，严重影响模型迭代效率

高质量数据匮乏

RAG检索需要高质量知识库支撑，但企业内部知识散落各处，难以整合利用

私域数据安全风险

企业内部敏感数据需要安全可控的处理环境，不能直接上传到公有云平台

现有方案的局限

🧩
工具碎片化

采集用A、清洗用B、标注用C，工具分散难以统一管理

🔗

    覆盖数据采集、管理、清洗、合成、标注、评估、知识生成全生命周期

  • 可扩展架构

    丰富的算子生态，支持自定义算子开发，满足各种业务场景

  • 开源社区

    支持大规模数据处理，分布式执行引擎，多租户隔离，安全可控

版本管理

代码示例 — 简单过滤算子

from ops.base_op import BaseOp

class LengthFilterOp(BaseOp):
    """过滤文本长度低于阈值的文档
    
    用途：清洗数据时过滤过短的无效文档
    参数：min_length - 最小文本长度阈值（默认100字符）
    """
    
    def execute(self, data: list[dict]) -> list[dict]:
        min_length = self.config.get("min_length", 100)
        return [
            d for d in data 
            if len(d.get("text", "")) >= min_length
        ]
继承 BaseOp 实现 execute 方法 从 config 获取参数

只需10行代码，即可实现一个功能完整的过滤算子，开发门槛极低

代码示例 — LLM增强算子

from ops.base_op import BaseOp
from ops.llms import LLMClient

class QualityEvaluatorOp(BaseOp):
    """使用LLM评估文档质量
    
    用途：自动评估训练数据质量，过滤低质量内容
    参数：model - 使用的LLM模型名称
    """
    
    def execute(self, data: list[dict]) -> list[dict]:
        client = LLMClient(self.config.get("model"))
        results = []
        for doc in data:
            score = client.evaluate(doc["text"])
            doc["quality_score"] = score
            results.append(doc)
        return results
集成LLMClient 批量处理数据 新增质量分数字段

LLM算子可调用大模型能力，实现智能化数据处理，大幅提升处理质量

算子开发最佳实践

性能优化策略

  • 批量处理优先 — 避免逐条处理，使用批量操作减少I/O开销，提升处理效率10倍以上
  • 异步并发执行 — 使用 asyncio 处理I/O密集型任务，充分利用Ray分布式能力
  • 缓存中间结果 — 对于重复计算（如Embedding生成），使用本地缓存避免重复调用
  • 分块处理大数据 — 超大数据集分块加载，避免内存溢出

错误处理规范

  • 异常捕获与记录 — 使用 try-catch 捕获异常，通过 logger 记录完整错误信息
  • 优雅降级策略 — 部分数据失败不影响整体流程，记录失败数据供后续处理
  • 超时机制 — 设置合理的超时时间，避免任务无限等待
  • 重试策略 — 对于网络请求等不稳定操作，实现自动重试机制

遵循最佳实践开发的算子，运行稳定、性能优异，适合生产环境大规模部署
